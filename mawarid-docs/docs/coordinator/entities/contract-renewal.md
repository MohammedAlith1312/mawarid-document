# Contract Renewal

The **Contract Renewal** page triggers workflow-based notifications to employees asking them to accept or reject their contract renewal.

## Page Type

- **Type:** Entity — Workflow

## Workflow Stages

`Draft` → `Sent` → `Accepted` / `Rejected`

## How It Works

1. Coordinator initiates a contract renewal for one or more employees.
2. A **notification message** is automatically sent to all targeted employees.
3. Employees receive the notification in the **Employee Portal** and respond with **Accept** or **Not Accept**.
4. The coordinator can track responses from this page.

> **Integration:** This page integrates with the **Employee Portal** for employee response collection.

## Notes

- Contract renewal notifications respect the employee's language preference.
- Coordinator can filter employees by project or nationality before sending.
