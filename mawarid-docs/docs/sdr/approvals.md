# Approvals

The **Approvals** page is used by the **General Manager** to review and act on submitted SDR requests awaiting approval.

## Views

| Tab | Description |
|---|---|
| **Pending** | Requests submitted by end users awaiting GM decision |
| **Completed** | Requests where GM has already approved or rejected |
| **All** | Full approval history |

## Approval Process

For each pending request, the GM:

1. **Reviews** the request:
   - Summary
   - Selected system (ERP / CRM / Portal / Integration)
   - Requirements list (user stories)

2. **Decision:**

| Action | Next Step |
|---|---|
| **Approve** | GM enters **Project Benefits** (Legal, Financial, other) and submits → Status: **IT Review** |
| **Reject** | GM enters rejection reasons → Status: **Rejected** (returned to requester) |

## Project Benefits Form

When approving, the GM fills in the Project Benefits section:

| Benefit Type | Description |
|---|---|
| **Legal** | Regulatory compliance, legal risk reduction |
| **Financial** | Cost savings, revenue impact |
| **Other** | Any additional strategic or operational benefits |

:::note
After GM submits the approval with project benefits, a popup shows the requester how many requests are ahead in the queue.
:::
