# IT — Requests

The **IT Requests** page allows the IT team to view and manage incoming SDR requests that have been approved by the General Manager and are now in the **IT Review** stage.

## Views

| Tab | Description |
|---|---|
| **Pending** | Requests currently awaiting IT review and estimation |
| **All** | Full history of all requests handled by IT |

## IT Review Process

For each request in the Pending tab, the IT team must:

1. **Review requirements** — Check if the requirements list is clear.
   - If **not clear**: Enter issues and dependencies, then enter the estimated duration.
2. **Enter Estimation** — Provide the number of development days required.
3. **Decision based on estimation:**

| Estimation | Next Step |
|---|---|
| **≤ 21 days** (Simple Development) | Schedule project → Status: **Approved** |
| **> 21 days** | Escalate to **Steering Committee** |

## Request Details

The request detail view includes:
- Summary and selected system (ERP / CRM / Portal / Integration)
- Requirements list (user stories with priorities)
- GM-entered Project Benefits (Legal, Financial, etc.)
- Issues and dependencies entered by IT
- Estimated development duration
